아파치 사용자계정 파일을 저장권한 설정

사용자계정으로 minami 하나를 생성후, 안에 웹폴더 www 를 생성하였습니다.
가상호스팅으로 minami.jinyweb.com 도메인 설정 하였습니다.

일부 웹페이지는 디렉토리를 생성하거나 파일을 생성저장하는데 아파치와 해당계정파일의 chown
권한이 차이가 있어서 정상적인 동작을 하지 않습니다.

ex) /home/minami/www/conf 디렉토리가 겨정의 minami 소유로 되어 있으면 아파지
웹이 해당 경로의 퍼미션이 틀려서 파일저장이 되지 않습니다.

/home/minami/www/conf 를 httpd.conf 의
user/group 의 wwwhtml로 소유권을 변경하면 잘 저장이 되고 동작합니다.

여러개의 사용자 계정으로 웹사이트를 운영할때 , 아파치에서 파일 읽고/쓰기를 할려면 권한을
어떻게 맞추어야 되는지 모르겠네요...
